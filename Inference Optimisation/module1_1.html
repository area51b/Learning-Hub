1 Why Does This Even Matter?
▾

When customers ask you about inference optimization - flash attention, disaggregation, KV cache, quantization - every single one of these concepts exists to solve a problem that only exists because of how Transformers work.

You can't properly explain why KV Cache matters if you don't know what a Transformer is doing internally. You can't explain prefill vs decode disaggregation without understanding the two phases of generation.

Think of it this way: If a customer asks "why does disaggregation help with latency?", you need to trace the answer back to the transformer. This module is that foundation.
2 What Is a Transformer?
▾

A Transformer is a type of neural network architecture introduced by Google in 2017. Today, it powers almost every major AI model - GPT-4, Claude, Gemini, LLaMA, Mistral. They're all Transformers.

Before Transformers
Models read text word-by-word, left to right
Slow - couldn't parallelize
Forgot context from earlier in the sentence
Couldn't handle long documents well
With Transformers
Reads ALL tokens simultaneously
Parallelizes massively across GPUs
Understands context across thousands of words
Foundation of all modern LLMs
Key insight: The "read everything at once" capability is called Attention - and it's the heart of why Transformers are powerful. You'll learn exactly what it does in Module 1.3.
3 The Analogy: A Super-Smart Translator
▾

Imagine you receive a very long email from a customer - 4 pages long. You need to write a reply.

Old AI approach (pre-Transformer)
You read the email word by word, and by the time you get to page 4, you've mostly forgotten what was on page 1. Your reply is based mostly on the last few lines.
Transformer approach
You read the entire email at once. You highlight relationships - "this question on page 4 connects to the constraint mentioned on page 1." You write a reply that considers ALL the context simultaneously.
Why this matters for inference: Since the Transformer reads everything at once, it generates massive amounts of intermediate data (called the KV Cache) that has to be stored in GPU memory. This is exactly why memory management is the #1 problem in inference optimization.
4 What Happens When You Type a Prompt
▾

When you type "Summarize this 10-page contract for me", here's the journey:

1
Tokenization
Your text is broken into tokens - small chunks of text (roughly 3/4 of a word each). A 10-page contract might be ~6,000 tokens.
2
Prefill Phase
The model processes ALL your input tokens at once. This is computationally intensive - like reading the whole contract in one sitting. This phase is compute-bound.
3
KV Cache Is Created
As the model reads your tokens, it saves key information in memory (the KV Cache). This is like taking notes so it doesn't have to re-read everything for each word it generates.
4
Decode Phase
The model generates the summary, ONE token at a time. Each new token looks back at the KV Cache to maintain context. This phase is memory-bound.
5
Output
Tokens are streamed back to you as words. You see the summary appear word by word.
This is your bridge to disaggregation: Prefill and Decode have completely different resource needs. Disaggregation splits them onto different hardware to optimize each independently. Now you know why!
5 3 Key Ideas to Remember
▾

If you remember nothing else from this module, remember these three things:

01
Transformers read everything at once
Unlike older AI, Transformers process all input tokens simultaneously. This gives them incredible context understanding - but creates big memory pressure.
02
There are two phases: Prefill and Decode
Prefill = reading your input (compute-heavy). Decode = generating the response token-by-token (memory-heavy). Almost every optimization technique targets one or both of these.
03
The KV Cache is the key bottleneck
The intermediate memory state saved during processing. It's what enables fast generation - but it's huge and fills up GPU memory fast. Flash Attention, Paged Attention, disaggregation - they all relate to this.
MODULE 1.1 - CHEAT SHEET
Transformer
Neural network architecture that reads all tokens at once using Attention
Token
A chunk of text (~3/4 word). Your prompt = many tokens.
Prefill
Phase 1 - model reads all input at once. Compute-bound.
Decode
Phase 2 - model generates output one token at a time. Memory-bound.
KV Cache
Memory structure storing intermediate state. Enables fast generation but consumes GPU memory.
Attention
The mechanism that lets the model relate any token to any other token. Core of the Transformer.
